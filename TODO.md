## 1. Переключатель отображения имени мода в списке: папка ↔ ModuleInfo Name
Сейчас в списке метка мода = **имя папки** (`mid.split('/').pop()` в `leafRow`,
app.js). В игре же мод показывается по полю **Name** из ModuleInfo (см. скриншоты:
папка `ShusRangers/ShuMM`, а в игре это `PolMM`). Нужен переключатель «Папка / Имя
из ModuleInfo» (по аналогии с уже существующим «По папкам / По разделам» = `tree_mode`).
- Данные уже есть: `module_card()`/`get_mod_info` дают `name`; для каталожных модов
  `catalog.json` теперь несёт имя варианта. Для дисковых — читать Name с диска
  (как `_section_of`/`_desc_of`), кэшировать.
- Добавить режим в `get_tree` (поле `display`/новый `label`), сегмент-кнопку в тулбаре,
  сохранять выбор в config (как `tree_mode`).
- Важно: для Pol*-папок (ShuMM-папка с Name=PolMM) это и даст «как в игре».

## 2. Нужна ли «База сборки» в настройках?
В «Настройки» есть выпадашка **«База сборки»** (`#setBase`, значения original/redux/
universe), на скрине пустая. При этом база и так **автоопределяется с диска**
(`detect_installed_base` по BASE_MARKERS) и показывается в шапке дерева. Похоже на
дубль/путаницу.
- Разобраться: где `profile['base']` реально используется (поиск по `profile.get('base'`,
  `setBase`, `prof_camp`). Кажется — только как метка лагеря для дисковых модов вне
  установленной базы + перенос профиля.
- Решить: убрать из настроек (оставить автоопределение) ИЛИ оставить как явный выбор
  для переносимой сборки, но пояснить смысл. Сейчас пустое поле сбивает с толку.

## 4. Проверка зависимостей/конфликтов ПРИ ВКЛЮЧЕНИИ мода (как в игре)
Сейчас включение в сборку (`onToggleClick` → `toggle_enabled`) ставит галочку без
проверки. В самой игре при подключении мода показывается каскад изменений и запрос
подтверждения (см. скриншоты):
- «При подключении мода будут выполнены изменения: + подключено DenDynamicBG,
  − отключено PolarPirateBG. Продолжить?» (зависимости тянутся, конфликтующие — гаснут)
- «− отключено Dr_Kles_Mod, EvoArmada. Продолжить?»
- «Невозможно выполнить, отсутствует необходимый мод — ShuKlissan!» (блок, если зависимости нет)
Нужно: при включении мода прогнать резолв и показать модалку с каскадом
(что подключится по зависимостям / что отключится по конфликтам), с подтверждением;
если обязательной зависимости нет ни в сборке, ни на диске/в каталоге — блокировать
с понятным сообщением.
- Кирпичи уже есть: `core.resolve_set` (замыкание depends + детект conflicts по каталогу),
  `check_disk_dependencies`. Конфликт/зависимость — по полю Name из ModuleInfo (Conflict/
  Dependence), матч по короткому имени. Сейчас resolve_set используется только при
  «Установить всю сборку» — надо подключить и к toggle отдельного мода.
- Решить уровень: на уровне СБОРКИ (профиль) и/или при записи в игру (ModCFG,
  `profile_to_modcfg`) — у игры это про активацию в ModCFG.
- UX: показывать каскад в модалке (можно переиспользовать стиль `#compatOverlay`),
  кнопки Продолжить/Отмена; «отключаемые по конфликту» — отдельным цветом.
