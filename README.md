# SR Mods Launcher

Лёгкий лаунчер для Space Rangers HD: ведёт список модов, скачивает и ставит их в папку
`Mods`, показывает даты обновления/установки, запускает игру. Профили и темизация.

> 🎮 **Новичку:** пошаговое руководство для игрока — в [ИНСТРУКЦИЯ.md](ИНСТРУКЦИЯ.md).
> Этот README — техническая справка (сборка, форки, темизация).

## Возможности (v0.9)

- **Профиль = переносимый набор.** Профиль хранит постоянно: **базу** набора (поле «База»: лагерь
  redux/universe/...), **набор подключённых модов** (как `ModCFG`), **путь к игре** (и задел на
  гит-форк агрегатора). Такой профиль можно отдать другому игроку. База **в папке игры** — это
  отдельный **факт** (определяется по диску), а не часть профиля; если базы профиля и диска не
  совпадают, в дереве будут **обе** — установленная и «добавить». См. колонки и синхронизацию ниже.
- **Подключение модов (ModCFG) ⟷ профиль.** Две новые колонки дерева: **«Подключен в игре»** (по
  `Mods/ModCFG.txt` — что игра реально грузит) и **«Подключен в профиле»** (набор `enabled` профиля).
  Подключение в профиле переключается **кликом по колонке** или через ПКМ-меню. Панель **«Подключение
  в игре (ModCFG)»**: **⟸ Считать из игры в профиль** (ModCFG → профиль) и **⟹ Записать профиль в
  игру** (профиль → `ModCFG.txt`, предупреждает о модах не на диске). Прочие строки и кодировка
  ModCFG сохраняются.
- **Дерево Лагерь → Пак → Мод.** В одном списке и установленное на диске (колонка «Установлен» с
  датой), и добавленное в набор (`➕ добавлен`). Моды **набора профиля**, которых ещё нет на диске,
  показываются строками `📥 доступен` / `⚠ недоступен` (по каталогу); ПКМ «⬇ Добавить в набор»
  ставит их в очередь скачивания. Переключатель **«Вид»**: группировка модов
  **по папкам** Mods или **по разделу** (`Section` из ModuleInfo). Фильтр **«Только требующие
  внимания»** скрывает `✅ установлен`. ПКМ по строке — открыть папку мода / обновить с сохранением
  правок. Кнопки развернуть/свернуть. Дисковые и профильные моды группируются единообразно по папке/разделу.
  В шапке — **бегущий итог очереди** «в наборе: N · ~X ГБ».
- **Очередь набора — сессионная.** Добавленное через ＋ — это очередь на установку; исчезает при
  ручном удалении, после установки (мод становится дисковым) или при перезапуске. **`－` убирает
  несколько выделенных** и **группу целиком** (лагерь/пак разворачивается в записи набора).
- **Раскладка установки = как в игре.** `install_route`/`install_target`: контент `Mods/` → в `Mods`,
  файлы базового пака (`Rangers.exe`, `matrix/`, `CFG/`, `DATA/`) → в **корень игры**; staging-обёртки
  (`*_unpacked/`, `.temp/`), Inno-плейсхолдеры (`{userdocs}`) и мусор инсталлятора (`.iss`,
  `.exe`-установщики, readme) — **пропускаются**. Полная установка лагеря идёт по порядку
  **база → паки/моды → фиксы паков → фиксы базы**.
- **Параллельная загрузка.** Части качаются в несколько потоков (общая `requests.Session` с пулом) —
  на канале с большим RTT/DPI один поток не насыщает полосу. Размер потоков — env `SRML_PARALLEL`
  (по умолч. 6).
- **Реальные размеры.** Перед крупной установкой — реальный объём (из `state/packs.json`,
  `code/asset/bytes` на пак) с пометкой про базовый пак.
- **Сохранение правок при обновлении (3-way merge).** Кнопка **`🔀 Обновить (сохранить правки)`**:
  если игрок руками менял файлы мода, при обновлении они **по возможности сохраняются**. Текст
  сливается построчно (diff3, чистый Python — без git); непересекающиеся правки — автоматически,
  настоящие конфликты — **с предпросмотром и выбором по каждому файлу** (оставить мой / взять новый /
  сохранить оба как `.srnew`). Работает и для модов набора, и для **дисковых** (из «Установлено в игре»).
  База 3-way не хранится байтами — снимок установки (`relpath→sha256`) лежит вне `Mods` в
  `~/.sr-mods-launcher/snapshots/`, содержимое подтягивается по sha. Можно выбрать несколько модов —
  обновятся по очереди.
- **Индексация модов на диске** (кнопка **`🗂`**, при первом запуске — предложение): локально, без
  скачивания, определяет базу и помечает моды `✅ знаком каталогу` / `❓ не в каталоге`. Инкрементально
  (sha файла переиспользуется по size+mtime) — быстро при повторе, учитывает добавленные/удалённые моды.
  **После установки лаунчером индекс пополняется сразу** — sha берётся из скачанных байт (`sha_sink`/
  `sha_cache`), без повторного чтения с диска; реиндекс не идёт «с нуля».
- **Установка с зависимостями** (`🧩`): **резолвит `Dependence`** из ModuleInfo (замыкание зависимостей),
  показывает конфликты (`Conflict`) и список к установке с подтверждением. «Установить весь набор`
  пропускает уже актуальные (идемпотентность). Код и ассеты — content-addressed части с Hugging Face
  (публично, без токена).
- **Определение установленной базы** по уникальным маркерам в `Mods/` (redux/original/universe).
- **Совместимость** (`🛡`): паки набора (одна база, фикс→родитель, сейвы) **+ незакрытые зависимости**
  (`Dependence`) установленных модов. Работает и при пустом наборе. Недостающие зависимости, которые
  есть в каталоге, можно **доустановить прямо из отчёта** («мост» — резолвит и ставит тем же
  механизмом, что `🧩`).
- **Прогресс и отмена:** индикатор `пак K/T · часть N/M · ↓ X.XX ГБ` — агрегатные гигабайты растут
  непрерывно (видно, что не зависло даже на крупной части). Кнопка **`✖ Отмена`** прерывает операцию;
  на время операции интерфейс блокируется. Чекбокс **«Подробный лог»** — краткий/детальный вывод.
  Сетевые запросы — с авто-повтором (устойчивость к обрывам).
- **Обновления** (`⟳`): для модов набора — сверка версии-хэша с каталогом.
- **Горячие клавиши** Ctrl+C/V/X/A работают в полях ввода при любой раскладке.
- Профили (переносимые наборы: база + подключённые моды + путь), запуск игры, выбор папки игры с
  автопоиском Steam, форки по URL дескриптора.

Запись очереди установки бывает трёх типов: `unit` (пак или один мод агрегатора), `camp` (весь
лагерь), `desc` (мод по дескриптору/форк-URL). Профиль дополнительно хранит постоянные поля:
`base` (выбранная база), `enabled` (подключённые моды, как ModCFG), `game_path`, `fork`. Очередь
установки (`mods`) — сессионная (см. ИНСТРУКЦИЯ.md), `enabled`/`base` — постоянные и переносимые.

**Проверка совместимости** (кнопка `🛡 Проверить совместимость`): по `state/packs.json`
агрегатора лаунчер проверяет набор — ровно одна **база** (пак с `Rangers.exe`), наличие
родительских паков для **фиксов** (`fix_parent`), несовместимость сейвов при смене базы,
обязательность base/fix-обновлений; плюс конфликты/зависимости модов. Только показывает.

**Форк мода:** опубликуй свой `<id>.json` дескриптор (те же поля; `chunk_index_url` может
указывать на твой HF/хост) где угодно и дай на него прямую ссылку в поле «URL форка».

## Запуск из исходников

```
pip install -r requirements.txt
python launcher.py
```

Зависимости минимальны: `tkinter` (входит в стандартный Python) и `requests`.

## Сборка .exe (Windows)

Требуется Python 3.8+ и PyInstaller.

```
pip install -r requirements.txt pyinstaller
build.bat
```

Либо вручную:

```
pyinstaller --onefile --windowed --name SRModsLauncher --add-data "theme.json;." launcher.py
```

Результат: `dist\SRModsLauncher.exe` — самодостаточный файл (Python и библиотеки внутри).

**Антивирус (ложное срабатывание):** `--onefile` exe часто помечается Chrome/Defender как
«вирус» (эвристика на самораспаковку PyInstaller). Это ложь. В Releases лежат **оба** варианта:
- `SRModsLauncher.exe` — готовый файл (двойной клик, Python внутри);
- `SRModsLauncher-source.zip` — исходники + `start_launcher.bat` (если антивирус мешает exe).

Папочная сборка (`--onedir`) флагуется реже; радикально решает подпись сертификатом (платно).

Замечания по сборке:
- `--windowed` — без консольного окна (GUI-приложение).
- `theme.json` встроен в exe как тема по умолчанию. Чтобы переопределить оформление,
  положи свой `theme.json` **рядом с .exe** — он имеет приоритет.
- Конфиг (`launcher_config.json`) и `profiles/` создаются рядом с .exe при первом запуске.
- Иконку можно добавить флагом `--icon app.ico`.

## Приватный репозиторий агрегатора

Если репозиторий с модами приватный, укажи **GitHub token** в поле настроек лаунчера
(или переменную окружения `GH_TOKEN`). Нужны права `repo` (чтение релизов и файлов).
Токен сохраняется в `launcher_config.json` рядом с лаунчером.

## Темизация (`theme.json`)

```jsonc
{
  "name": "Space Dark",
  "bg": "#0b0e17", "panel": "#141a26", "fg": "#d6e1ff", "muted": "#7f8db0",
  "accent": "#3a6df0", "accent_fg": "#ffffff",
  "tree_bg": "#0f1521", "tree_sel": "#1f3a6e",
  "font_family": "Segoe UI", "font_size": 10, "mono_family": "Consolas",
  "banner": ""            // путь к PNG-баннеру (опц.), рисуется в шапке
}
```

Меняешь цвета/шрифты/баннер → перезапускаешь лаунчер. Можно оформить под стиль игры.
