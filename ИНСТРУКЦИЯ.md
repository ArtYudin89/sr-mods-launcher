# SR Mods Launcher — инструкция для игрока

Пошаговое руководство для того, кто видит лаунчер впервые. Лаунчер помогает выбирать наборы
модов для **Space Rangers HD: A War Apart**, скачивать и ставить их в игру, показывает, что уже
установлено, и следит за обновлениями.

---

## 0. Что нужно и как запустить

- Установленная игра **Space Rangers HD** (с файлом `Rangers.exe`).
- **Python 3.8+** (с [python.org](https://python.org)) — лаунчер запускается из исходников.
- **GitHub-токен** — если репозиторий с модами приватный (по умолчанию он приватный). Как
  получить — в конце инструкции.

**Запуск:** распакуй `SRModsLauncher-source.zip` из раздела Releases и **двойным кликом** запусти
**`start_launcher.bat`**. Установка не нужна; рядом появятся файлы настроек.

> ⚠ Готовый `.exe` мы не раздаём: упаковщик PyInstaller часто ложно срабатывает у антивируса.
> Запуск из исходников (`start_launcher.bat`) этого лишён.

---

## 1. Первый запуск — папка игры

Блок **«Настройки»**:
1. **Папка игры** — лаунчер пытается найти Steam-папку сам. Если не нашёл — нажми **«Обзор»** и
   укажи **папку** игры (где лежит `Rangers.exe`), не сам exe.
2. Если репозиторий приватный — вставь **GitHub token** (поле справа).
3. **Репозиторий** менять не нужно (по умолчанию основной).

Кнопка **«📂 Папка модов»** открывает `Mods` игры. **«▶ Запустить игру»** — запускает `Rangers.exe`.

---

## 2. Профиль = набор модов

**Профиль** — сохранённый набор. Кнопки **Новый / Сохранить / Удалить**. Можно держать несколько
наборов и переключаться.

---

## 3. Дерево модов

Главный список — дерево **Лагерь → Пак → Мод**. В одном списке и то, что **уже стоит** в игре, и
то, что ты **добавил** в набор. Колонки:
- **Тип** — лагерь / пак / мод / форк.
- **Статус** — `✅ установлен` (есть на диске), `⬆ обновление` (вышел апдейт), `➕ добавлен`
  (в наборе, ещё не установлен).
- **Установлен** — дата (есть дата = мод физически на диске).

Над списком:
- **Вид** — как группировать установленные моды:
  - **По папкам** — по папке внутри `Mods` (напр. `redux / OtherMods / DrKlesMod`).
  - **По разделу** — по полю `Section` из мода (напр. `redux / Солянка / LeoMMOnlyUnique`); у кого
    раздел не указан — группа «Не указан».
- **Только требующие внимания** — скрывает `✅ установлен`, оставляя обновления и добавленное.
- Кнопки: **＋** добавить, **－** убрать, **↑/↓** порядок, **⊞/⊟** развернуть/свернуть всё,
  **⟳** проверить обновления. У кнопок есть подсказки при наведении.
- **Правый клик по строке** — открыть папку мода или папку `Mods`.

---

## 4. Добавить моды (кнопка ＋)

Окно «Добавить», источник:
- **Из репозитория** — три каскадных поля:
  - **Лагерь** (обязательно). Пока не выбран — остальные недоступны.
  - **Пак** — по умолчанию **«— весь лагерь —»** (добавит все паки лагеря). Выбери конкретный пак,
    чтобы открыть выбор мода.
  - **Мод** — при выбранном паке по умолчанию **«— весь пак —»**; можно выбрать один мод.
- **По ссылке (форк)** — прямая ссылка на дескриптор `<id>.json` чужого форка мода.

То есть глубина выбора задаёт, что добавится: весь лагерь / весь пак / один мод.

---

## 5. Три правила набора

1. **Ровно одна база** (пак с `Rangers.exe`: Редакс / Оридж / Юнивёрс) — смешивать нельзя.
2. **Фикс** ставится только на свою базу/пак (фикс — это частые мелкие обновления того же пака).
3. **Сейвы привязаны к базе** — посреди партии базу менять нельзя (новую игру — можно).

**🛡 Проверить совместимость** (справа) покажет: `⛔` две базы, `⚠` нет базы / фикс без родителя,
`💾` смена базы ломает сейвы, `✅` база уже установлена. Лаунчер определяет установленную базу
по файлам на диске.

---

## 6. Установка и запуск

Блок **«Действия»**:
- **🧩 Установить набор (с зависим.)** — основной способ: ставит набор и подтягивает зависимости.
- **⬇ Установить выбранный / все** — точечно или всё.
- **🗑 Очистить Mods** — ⚠ **удаляет ВСЁ** из папки `Mods` игры (с подтверждением). Используй
  только для чистого старта.

Внизу справа — прогресс и лог. Затем **▶ Запустить игру**.

---

## 7. Обновления

Кнопка **⟳** сверяет версии с репозиторием. Где есть апдейт — статус **`⬆ обновление`**;
переустанови такой мод/набор. Обновления базы и фиксов ставить **обязательно**.

---

## GitHub-токен (для приватного репозитория)

1. **https://github.com/settings/tokens?type=beta** → **Generate new token** (fine-grained).
2. Имя любое; **Repository access → Only select repositories →** репозиторий с модами.
3. **Permissions → Contents → Read-only**. Сгенерировать, скопировать, вставить в поле
   «GitHub token» лаунчера.

Токен хранится локально в `launcher_config.json`. Никому его не показывай.

---

## Если что-то не так
- Пусто в каталоге / нет лагерей при добавлении — проверь токен и нажми **⟳**.
- Сменил базу — пропали сейвы: это ожидаемо, начни новую партию.
- Настройки и профили лежат рядом с `start_launcher.bat` (`launcher_config.json`, `profiles/`).
